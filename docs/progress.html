Visual progress summary

Roadmap order and implementation status in one view.

Mycel is still in the narrow first-client lane. The immediate goal is to finish the last shared-core closure work while widening the reader surface only through deterministic, replay-backed accepted-head inspection and rendering.

  1. Close M1 malformed-field depth and canonicalization debt.
  2. Finish M2 replay, rebuild, and merge-authoring closure.
  3. Expand M3 reader workflows carefully on top of the accepted-head inspection and render base.

Milestone Timeline

The roadmap is sequential: each later phase assumes the earlier one is already stable. Right now, the center of gravity is still the end of M1 and the closure of M2.

Implementation Matrix

This matrix compresses the checklist into section-level status so we can see what is substantively done, what is only partial, and what still belongs to later phases.

Area Status Milestone Current read
1. Repo and Build Setup Mostly done M1 Only the shared canonical JSON utility remains open.
2. Object Types and IDs Partial M1 All required v0.1 families are typed; malformed field-shape depth, semantic-edge closure, and role modeling remain.
3. Canonical Serialization and Hashing Partial M1 Core rules and reproducibility coverage exist; shared canonical utility reuse for state_hash and wire remains open.
4. Signature Verification Partial M1 Object signature rules are mostly present, and signature-edge verify smoke coverage is broader; wire-envelope verification is not.
5. Patch and Revision Engine Mostly done M2 Replay and state_hash are in place; the patch-op base is strong.
6. Local State and Storage Mostly done M2 Store ingest, rebuild, and indexes exist; policy separation still remains.
7. Wire Protocol Not started M4 Canonical wire envelope and message validation remain future work.
8. Sync Workflow Not started M4 First-time and incremental sync are still future work.
9. Views and Head Selection Mostly done M3 The deterministic selector core and named fixed-profile selection exist; dual-role closure remains.
10. Merge Generation Partial M2 Replay verification is present, but local merge authoring is not built.
11. CLI or API Surface Partial M2 / M3 Verification, authoring, reader inspection/render, and minimal governance publish/inspect exist; sync remains open.
12. Interop Test Minimum Partial M1 / M2 Fixture isolation, reproducibility, and smoke coverage exist, but several normative wire and replay checks remain.
13. Ready-to-Build Gate Partial Whole plan Replay, head selection, and rebuild are green; parse, wire sync, and merge generation are not.
