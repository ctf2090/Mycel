公开进度摘要

用一页看懂 Roadmap 顺序和 implementation 状态。

Mycel 目前仍处在窄版 first-client lane。眼前目标是先收掉最后一批 shared-core closure，同时让 M4 保持窄范围，聚焦在已验证 wire-session reachability、最小 peer-sync closure，以及建立在已落地 optional sync flows 之上的更广 interop proof。

  1. 收掉 M1 的 malformed-field depth 和 canonicalization debt。
  2. 完成 M2 的 replay、rebuild 和 merge-authoring closure。
  3. 在 accepted-head inspect/render 和命名 profile 基础上，谨慎扩展 M3 reader workflows。
  4. 让 M4 保持窄范围，把 peer-store sync proof 往更广 interop closure 与未来 production replication behavior 收敛。

Implementation Matrix

这张表把 checklist 压缩成 section-level 状态，让我们快速看到哪些部分已经大致完成、哪些仍然是 partial、哪些仍属于后续 phases。

Area Status Milestone Current read
1. Repo and Build Setup Mostly done M1 只剩 shared canonical JSON utility 还没有收尾。
2. Object Types and IDs Partial M1 所有必需 v0.1 families 都已经 typed；剩余工作是 malformed field-shape depth、semantic-edge closure 和角色建模。
3. Canonical Serialization and Hashing Partial M1 核心规则和 reproducibility coverage 已经存在，而且 canonical helper reuse 更集中；state_hash 和 wire 仍然缺完整 shared utility reuse。
4. Signature Verification Partial M1 / M4 Object signature rules 已到位，通用 wire-envelope signature verification 也已存在，OBJECT body 的 hash / object_id 重算现在已接进 wire validation，而且 capability-gated optional messages 也已受控；剩下的是更完整的 shared-helper convergence 与更广 interop proof。
5. Patch and Revision Engine Mostly done M2 Replay 和 state_hash 已经存在；patch-op base 已经很稳。
6. Local State and Storage Mostly done M2 Store ingest、rebuild 和 indexes 已经存在；local transport/safety separation 还没有完成。
7. Wire Protocol Partial M4 Canonical wire-envelope parsing、field validation、sender checks、session sequencing/head tracking、reachability gating、store-backed bootstrap、OBJECT body verification，以及 capability-gated optional messages 现在都已存在于 mycel-core；更完整的 peer interop 仍未完成。
8. Sync Workflow Partial M4 Peer-store 驱动的 first-time 与 incremental sync pull flow 现在已能 verify/store requested objects，并覆盖 snapshot-assisted catch-up 与 announced-view fetching，且 WireSession 可从已验证 store state bootstrap；更广的 peer interop 仍待完成。
9. Views and Head Selection Mostly done M3 Deterministic selector core、命名 fixed-profile selection、具备 editor/view admission 感知的 inspect/render flows，以及 head inspection 中的 bounded viewer score channels 已经存在；更广泛的 governance persistence 与最终的 dual-role closure 仍待完成。
10. Merge Generation Partial M2 Replay-based verification 和保守型 local merge authoring 已经存在，涵盖 structural move/reorder、新 parent reparenting、简单 composed parent-chain coverage，以及更广的 nested structural matrix，但更丰富的 nested/reparenting conflict cases 仍然会回落到 manual curation。
11. CLI or API Surface Partial M2 / M3 / M4 Verification、authoring、保守型 merge authoring、具备 editor-admission 感知的 reader inspection/render、governance inspect/list/publish、persisted governance index query surfaces、transcript-backed sync pull，以及内部 peer-store sync 都已存在，并包含 optional snapshot/view flows；更广的 replication behavior 仍未完成。
12. Interop Test Minimum Partial M1 / M2 / M4 Fixture isolation、reproducibility、更严格的 parser/replay smoke coverage、直接的 wire-envelope/signature/session tests、peer-store 的 first-time / incremental sync proofs、optional-message coverage，以及 simulator positive-path smokes 已存在，但更广的 peer-interop cases 仍未完成。
13. Ready-to-Build Gate Partial Whole plan Replay、head selection、rebuild、保守型 merge authoring，以及 peer-store sync proofs 已经绿了；parse closure、更广的 peer interop，以及 production replication behavior 还没有完成。
