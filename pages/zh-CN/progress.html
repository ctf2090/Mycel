公开进度摘要

用一页看懂 Roadmap 顺序和 implementation 状态。

Mycel 目前仍处在窄版 first-client lane。`M2` 在当前窄版 replay/storage/rebuild 范围内已经完成收口， 因为 richer mixed content/metadata competing-branch classification、多文档 index-loss 后 rebuild proof，以及 richer metadata multi-variant rebuild/reporting proof 都已落地；因此现在的 active lane 转向更广的 M3 governance persistence、更完整的 governance tooling、reader-facing profile ergonomics，以及最后的独立 dual-role 收尾，再加上剩余的 M4 session/capability/error-path interop proof；而更广的 pre-HELLO / pre-root / pre-MANIFEST gating、 HEADS 先于 MANIFEST 建立 sync roots、 HEADS replace=true 后的 stale root/dependency、stale snapshot 与 stale object WANT rejection、unadvertised WANT、unrequested OBJECT，以及 missing-BYE warning handling 也都已有 proof。

  1. 维持 M2 在当前窄版 replay/storage/rebuild 范围内的已关闭状态，并以 richer mixed content/metadata competing-branch rebuild/reporting proof 作为基线。
  2. 在 accepted-head inspect/render、更清楚的 available-profile 和 profile-error feedback、editor-admission-aware profile base、bounded viewer score surfaces、初始 governance inspect/list/publish/current surfaces，以及 persisted governance relationship 与 current-governance summaries 之上，谨慎扩展 M3 的 reader-plus-governance workflows，同时把更广的 governance persistence、更完整的 governance tooling、超出当前初步打磨的 profile ergonomics，以及最后的独立 dual-role 收尾明确保留下来。
  3. 让 M4 保持窄范围；现在 sender-validation、更广的 pre-HELLO / pre-root / pre-MANIFEST gating、HEADS 先于 MANIFEST 的 sync-root setup、HEADS replace=true 后的 stale root/dependency、stale snapshot 与 stale object WANT rejection、unadvertised WANT、unrequested OBJECT、ERROR-before-HELLO acceptance、explicit ERROR-only、unreachable WANT，以及 messages-after-BYE / missing-BYE coverage 也都已有 proof，接下来把 peer-store sync proof 往剩余的 session/capability/error-path interop closure 收敛。

里程碑时间线

这份 roadmap 是串行推进的：后面的 phase 都默认前面的 phase 已经稳定。现在的重心是维持 M2 在当前窄版范围内的已关闭状态、M3 的谨慎扩展，以及剩余的 M4 互操作 proof。

早期部分完成

  • Bundle 和 store rendering，并保留 ancestry context
  • 命名 fixed-profile reading，并带有更清楚的 available-profile 与 profile-error feedback
  • 具备 editor-admission 感知的 inspect/render flows
  • head inspection 中的 bounded viewer score surfaces
  • Filtered governance inspect/list/publish workflows
  • Persisted governance reverse indexes
  • 在 view inspect / view list 里可见的 persisted governance relationship summaries
  • 在 view current 里提供的 per-document current-governance summaries
  • 剩余工作：更广的 governance persistence、更完整的 governance tooling、超出当前初步打磨的 profile ergonomics，以及最后的独立 dual-role 收尾
全栈

  • Capability-gated 的 SNAPSHOT_OFFER / VIEW_ANNOUNCE
  • localhost multi-process、re-sync idempotency、depth-N catchup 与 partial-doc selective sync proofs
  • 更广的 pre-HELLO / pre-root / pre-MANIFEST gating
  • HEADS 先于 MANIFEST 的 sync-root setup，以及 HEADS replace=true 后的 stale root/dependency、stale snapshot 与 stale object WANT rejection
  • unadvertised WANT 与 unrequested OBJECT rejection
  • unknown-sender 与 HELLO sender-identity mismatch rejection
  • ERROR-before-HELLO acceptance，以及 explicit ERROR-only 与 unreachable WANT rejection proof
  • messages-after-BYE rejection 与 missing-BYE warning proof
  • 剩余工作是更广的 session/capability/error-path interop proof
