公開進度摘要

用一頁看懂 Roadmap 順序與 implementation 狀態。

Mycel 目前仍在窄版 first-client lane。眼前目標是先收掉最後一批 shared-core closure，同時讓 M4 維持窄版範圍，聚焦在 transcript-backed sync pull、已驗證 wire-session reachability，以及最小 peer-sync closure。

  1. 收掉 M1 的 malformed-field depth 與 canonicalization debt。
  2. 完成 M2 的 replay、rebuild 與 merge-authoring closure。
  3. 在 accepted-head inspect/render 與具名 profile 基礎上，謹慎擴張 M3 reader workflows。
  4. 讓 M4 維持窄版範圍，把 transcript-backed first-time / incremental sync pull 往 minimal peer-driven sync 收斂。

Implementation Matrix

這張表把 checklist 壓成 section-level 狀態，讓我們快速看到哪些區塊已大致完成、哪些仍屬 partial、哪些仍在後續 phases。

Area Status Milestone Current read
1. Repo and Build Setup Mostly done M1 build/test 基礎已穩定，shared canonical helper ownership 也正收斂到獨立的 core module；完整 utility closure 仍未收尾。
2. Object Types and IDs Partial M1 所有必要 v0.1 families 都已 typed，且 parser / verify / CLI dependency-proof coverage 更完整；剩餘工作是 malformed field-shape depth、剩餘 semantic-edge closure 與角色建模。
3. Canonical Serialization and Hashing Partial M1 核心規則與 reproducibility coverage 已存在，且 canonical helper ownership 已更集中到 shared core module；replay 衍生的 `state_hash` 與 wire 仍缺最後的共用化收尾。
4. Signature Verification Partial M1 / M4 Object signature rules 已到位，通用 wire-envelope signature verification 也已存在，而 OBJECT body 的 hash / object_id 重算現在已接進 wire validation；剩下的是更完整的 shared-helper convergence 與 peer-driven sync plumbing。
5. Patch and Revision Engine Mostly done M2 Replay 與 state_hash 已存在；dependency verification、wrong-type 與 multi-hop ancestry proof、sibling declared-ID determinism，以及 render 路徑的 ancestry context 都更完整。
6. Local State and Storage Mostly done M2 Store ingest、rebuild 與 indexes 已存在；rebuild smoke 現在也會在摘要回報中保留 nested ancestry context，但 local transport/safety separation 尚未完成。
7. Wire Protocol Partial M4 Canonical wire-envelope parsing、field validation、sender checks、session sequencing/head tracking、reachability gating、store-backed bootstrap，以及 OBJECT body verification 現在都已存在於 mycel-core；capability-gated optional messages 與更完整的 peer interop 仍未完成。
8. Sync Workflow Partial M4 Transcript-backed 的 first-time 與 incremental sync pull flow 現在已能 verify/store requested objects，且 WireSession 可從已驗證 store state bootstrap；最小 peer-driven sync 與 optional flows 仍待完成。
9. Views and Head Selection Mostly done M3 Deterministic selector core、具名 fixed-profile selection，以及具備 editor-admission 感知的 inspect/render flows 已存在；dual-role closure 尚未完成。
10. Merge Generation Partial M2 Replay-based verification 與保守型 local merge authoring 已存在，涵蓋 structural move/reorder、新 parent reparenting、簡單 composed parent-chain coverage，以及更廣的 nested structural matrix，但更豐富的 nested/reparenting conflict cases 仍會落回 manual curation。
11. CLI or API Surface Partial M2 / M3 / M4 Verification、authoring、保守型 merge authoring、具備 editor-admission 感知的 reader inspection/render、governance inspect/list/publish、persisted governance index query surfaces，以及 transcript-backed sync pull 都已存在；peer-driven sync 仍未完成。
12. Interop Test Minimum Partial M1 / M2 / M4 Fixture isolation、reproducibility、更嚴格的 parser/replay smoke coverage、直接的 wire-envelope/signature/session tests，以及 transcript-backed 的 first-time / incremental sync-pull smokes 已存在，但 peer-driven sync 與 optional-message cases 仍未完成。
13. Ready-to-Build Gate Partial Whole plan Replay、head selection、rebuild、保守型 merge authoring，以及 transcript-backed sync-pull proofs 已綠；parse closure 與 peer-driven wire sync 尚未完成。
