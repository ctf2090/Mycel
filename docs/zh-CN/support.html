支持可持续的历史与文化保存基础设施

在保存模型真正围绕现实场景收紧之前，和 Mycel 一起推进它。

我们正在寻找已经面对历史文本、评注传统、档案治理或长期参考文本集合问题的设计伙伴、 资助方和早期合作伙伴。现在的支持会直接帮助我们完成窄版 first client core： verification、replay、可重建存储，以及面向长期文化材料、按规则导出的默认阅读结果。

开始交流 阅读资助概念说明

我们想合作的对象

设计伙伴

真正面对历史文本或文化资产工作流的档案机构、维护者或团队，可以一起塑造第一条实用 profile 和 client 路径。

资助方

关注开放基础设施、文化遗产保存或公共利益技术，同时在意可长期保存、可审计知识系统的资助方。

早期合作者

希望用具体保存 use case 压测 Mycel 的研究者、维护者、档案工作者或机构。

支持会加速什么

协议收敛

shared typed parsing、canonicalization，以及更清晰、适用于长期文本与参考文本集合的 profile 边界。

Deterministic 验证

replay verification、accepted-head selection、negative coverage 和 fixture-backed checks，支撑保存工作流。

First Client 就绪

可重建 local storage、simulator-backed workflows，以及更窄但更实际、面向文化文本治理的 interoperable client 目标。

为什么是现在

Mycel 还足够早，合作伙伴的输入仍然能影响第一条 profile 和第一版 client 的方向； 但它也已经足够具体，外部支持投进去的是实打实的工程工作，而不是抽象的协议探索。
